# 升級計畫存檔

這裡放的是每個大版本開工前寫的計畫書原稿。**留著是因為它們是工程過程的證據**——
每一份都寫了「當時看到什麼、為什麼那樣取捨、明確決定不做什麼」，那些判斷紀錄
比最後的程式碼更能說明專案是怎麼長成現在這樣的。

放進 `docs/plans/` 而不是留在根目錄，單純是門面問題：進 repo 第一眼該看到 README，
不是三份計畫書。
