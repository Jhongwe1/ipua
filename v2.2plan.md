## 0. 一句話目標

整個網站的外殼改成 **chatgpt.com 登入後** 的樣子：左側固定側邊欄（可收合）、黑色預設主題、預設英文；`/playground` 就是「聊天主頁」，其他頁面（新聞、文章、relay、設定…）顯示在右側內容區。原有功能全數保留，ChatGPT 有但我們沒有的功能，按鈕直接不放。

## 3. 色票與字體（新外殼設計 tokens）

以 chatgpt.com 實際值為基準（截圖校對後可微調）：

```
深色（預設）                          淺色
--bg        #212121  主內容區底        #ffffff
--sb-bg     #171717  側邊欄底          #f9f9f9
--elev      #2f2f2f  輸入框/彈窗/懸浮   #f0f0f0（輸入框）/#ffffff（彈窗）
--fg        #ececec  主文字            #0d0d0d
--muted     #b4b4b4  次要文字          #5d5d5d
--sub       #8f8f8f  更淡的字          #8f8f8f
--line      rgba(255,255,255,.10) 分隔線   rgba(0,0,0,.10)
--hover     rgba(255,255,255,.06) 列 hover rgba(0,0,0,.05)
--accent    #ececec  送出鈕底(白底黑箭) #0d0d0d（黑底白箭）
```

- 字體維持系統字（-apple-system / Segoe UI / 微軟正黑），不引外部字型（CSP 也不允許）。
- 圓角語彙照 ChatGPT：側欄列 8px、輸入框 24~28px（膠囊）、彈窗 16px、訊息氣泡 18px。
- 主題切換維持現有三段式（日／夜／自動），**預設改為夜**；語言維持中英切換，**預設改為英文**（有 localStorage 記錄者以記錄為準，不影響老訪客的既有偏好）。

## 4. 新外殼版面規格

### 4.1 桌機（≥840px）

```
┌─────────────┬──────────────────────────────────────────┐
│ [☰收合]      │  Chat ∨（聊天頁=模型選單；其他頁=頁名）    … EN ☾ │
│ ✎ New chat  │                                          │
│ API relay   │                                          │
│ ▸ Tools     │              主內容區                     │
│ ▸ Content   │   （聊天頁：歡迎語＋訊息流＋輸入框；        │
│ ──────────  │     其他頁：原本的頁面內容）               │
│ History     │                                          │
│  對話1  …   │                                          │
│  對話2  …   │                                          │
│             │                                          │
│ [頭像] 名字  │                                          │
│  聯絡我(鈕) │                                          │
└─────────────┴──────────────────────────────────────────┘
```

- 側邊欄**常駐**、寬約 260px，左上角一顆收合鈕（ChatGPT 的「側欄開關」圖示）；收合後整條滑走、內容區佔滿，再按展開。狀態記 localStorage。
- 左上角**不放**任何 logo／站名圖標（站長指定）。
- 右上角控制列：黑白切換、中英切換；**未登入**時多一顆「Sign in」。頭像**不在**右上角（移到側欄左下，跟 ChatGPT 一樣）。

### 4.2 側邊欄內容（由上到下）

1. **New chat**（✎ 圖示）→ 到 `/playground` 開新對話（聊天頁內＝清空當前對話，不重載）
2. **選單項目**（沿用 D1 `menu` 表，編輯選單功能不變，對應規則見 4.3）
   - API relay → `/relay`
   - VPN → `/vpn`（維持現有隱形邏輯：管理員或被批准 vpn 的人才看得到）
   - **Tools** ▸（可展開，預設收合）：IP Lookup `/ip`、UA Lookup `/ua`
   - **Content** ▸（可展開，預設收合）：News `/news`、Articles `/articles`
3. **分隔線**
4. **History**（預設展開）＝ playground 歷史對話，跨頁面都看得到
   - 每列 hover（手機為常駐）「…」選單：**Rename／Delete**
   - 在聊天頁點對話＝當頁切換；在其他頁點對話＝導向 `/playground` 並打開該筆
   - 未登入或未獲批准 playground：History 區塊整個不顯示
5. **左下角固定區**：
   - 已登入：頭像＋名字（**不放** Free plan 字樣）＋「**聯絡我**」鈕（原 Upgrade 位置；連 settings 的 contact_url，沒設定就整顆隱藏）
   - 未登入：整塊變成一顆「**Sign in**」按鈕
   - 點頭像 → 上彈選單（照 `151344.png` 版面）：一般會員只有 **Log out／Log out everywhere**；管理員上面多五項（決策 3）

### 4.3 D1 選單相容規則（編輯選單功能不砍）

現有 `menu` 表是「section 標題＋link」的扁平清單。新外殼的對應規則：

- **第一個 section**（預設「服務」）：標題不顯示，裡面的連結**平鋪**在 New chat 下方（其中指向 `/playground` 的連結不重複顯示 — New chat 已經是它）
- **第二個以後的 section**：整組變成**可展開群組**（標題＝群組名，預設收合，展開狀態記 localStorage）
- 空表＝內建預設，行為同上。管理員照樣能在 /settings 編輯選單（✎ 編輯模式改為在側欄原地編輯，功能不變）

### 4.4 聊天頁（/playground）

- 頂部：「**Chat ∨**」下拉＝模型選單（取代現在的 `<select>`；照 `151018.png` 樣式：模型名＋勾選）。今日用量小字（`今日 3 / 20`）保留，放在頂列右側。
- 右上角「…」→ **只留 Delete**（刪除當前對話；沒開對話時不顯示這顆）
- 空對話狀態：置中歡迎語「**How can I help?**」（中文「有什麼我能幫上的？」）＋置中輸入框（照 `150831.png`）；發出第一句後輸入框落到底部（照 `150908.png`）
- 輸入框（照 `151714.png`）：膠囊形、左「＋」鈕（按下 flash「Feature not available / 功能未開放」）、右圓形送出鈕（串流中變停止鈕，行為沿用現況）
- 訊息流、Markdown 渲染、思考過程（think 區塊）、串流、demo 模式橫幅、額度 429 的「聯絡我」鈕 — **邏輯全部沿用**，只換皮
- 現有聊天卡片外框（`.pg` 那個帶陰影的框）**移除**：聊天直接鋪滿內容區，跟 ChatGPT 一樣

### 4.5 其他頁面（news/articles/relay/vpn/members/settings/logs/admin/api-docs/p/slug）

- 一律進新外殼：左側同一條側邊欄，右側內容區置中欄（閱讀頁 ~720px、管理頁 ~820px，同現況）
- 頁面內部元件（卡片、表格、表單、彈窗）**第一階段不重寫**，改吃新色票變數自動變深色；視覺明顯突兀的個別微調
- 頂部左側顯示頁名（原 h1），無模型下拉

### 4.6 /ip、/ua（public/index.html 靜態 SPA）

- 重寫 index.html 的外殼部分套新設計（側邊欄改為與 SSR 外殼同結構同樣式）
- ⚠ 它的 inline script 走 `public/_headers` 的 sha256 CSP：改完必跑 `tools/check-csp.mjs` 重算並更新雜湊，否則整頁腳本被 CSP 擋掉

### 4.7 手機版（<840px）

- 側邊欄變抽屜（照 `151332.png`）：左上 ☰ 開、overlay 點擊關；抽屜內容＝桌機側欄全套（含左下頭像區）
- 頂列：☰＋「Chat ∨」＋右側「…」（照 `151320.png`／`151705.png`）；黑白／中英切換鈕在手機收進哪裡 → **維持頂列右側**（兩顆小圓鈕，不另設選單，避免多一層）
- 輸入框吸底、鍵盤彈出不縮放（維持現有 16px 鐵則與 viewport 處方）

### 5.2 分階段（每階段結尾：format:check → lint → typecheck → test 四項全綠才前進）

| 階段 | 內容 | 驗收 |
|------|------|------|
| **A. 新外殼核心** | site.ts 新版面＋色票＋側欄（含收合、群組展開、D1 選單相容）＋account.js 重寫（頭像左下＋選單）＋預設夜／英 | 所有 SSR 頁面進新殼；桌機＋手機檢查；登入/未登入/管理員三種身分檢查 |

| **C. History 整合** | 側欄 History（跨頁）＋「…」Rename/Delete＋新對話行為 | 聊天頁內切換不重載；他頁點擊正確跳轉 |

| **E. 收尾** | 全站順走各頁微調、e2e 更新、文件（README 截圖/敘述若提到 UI）、ASSET_V/VERSION | 四項 CI＋e2e 綠；`npm run deploy` 上線；實機（桌機＋手機）驗收 |

### 5.3 風險與對策

- **外殼 DOM 大改會弄壞 adminbar 的選單編輯**：A 階段就把 `#sbMenu`/`#sbAdmin` 的掛點語意保留（換版面不換 id），編輯模式實測後才收 A。
- **CSP**：新殼所有 inline script 照舊走 `data-nonce` 蓋章制；index.html 走 sha256＋工具重算。不引任何外部資源。
- **快取**：assets 有 4 小時邊緣快取，靠 `ASSET_V` 單一常數破快取（v2.1 已收斂，照規矩調大即可）。
- **手機鍵盤**：沿用現有 viewport＋16px 處方，新輸入框元件必須實機（Android Chrome＋iOS Safari）各測一次。
- **回滾**：每階段獨立 commit；上線後發現嚴重問題可 `git revert` 該階段再 deploy。
